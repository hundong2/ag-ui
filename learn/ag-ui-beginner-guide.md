# AG-UI Beginner Guide

AG-UI는 **AI Agent와 사용자 UI를 연결하는 이벤트 기반 프로토콜**입니다.
처음 시작하는 분을 위해 "어디에 쓰는지"와 "어떻게 쓰는지"를 짧게 정리했습니다.

## 1) AG-UI는 어디에 사용되나요?

AG-UI는 아래 같은 상황에서 자주 사용됩니다.

- **챗봇/코파일럿 UI**: 답변이 한 번에 아니라 스트리밍으로 보일 때
- **에이전트 상태 동기화**: 서버의 상태 변경을 UI에 실시간 반영할 때
- **툴 호출 결과 표시**: 에이전트가 어떤 툴을 실행했는지 UI에서 보여줄 때
- **다양한 프레임워크 연동**: LangGraph, CrewAI 등과 UI를 같은 방식으로 연결하고 싶을 때

한 줄로 요약하면:

> "에이전트 백엔드에서 발생하는 이벤트를, 프론트 UI가 표준 방식으로 받게 해주는 연결 규칙"입니다.

## 2) 가장 빠른 시작 방법

루트 README 기준으로 가장 쉬운 시작 명령어는 아래입니다.

```bash
npx create-ag-ui-app my-agent-app
```

그다음 순서:

1. 생성된 프로젝트 실행
2. AG-UI 이벤트 스트림 확인
3. 메시지/툴/상태 이벤트가 UI에 어떻게 반영되는지 확인

추가로 실전 예시는 Dojo에서 확인할 수 있습니다.

- Dojo: https://dojo.ag-ui.com/
- Quickstart: https://docs.ag-ui.com/quickstart/applications

## 3) 어떻게 동작하나요? (초간단)

AG-UI 흐름은 보통 아래와 같습니다.

1. 사용자가 UI에서 요청 입력
2. 에이전트 서버가 요청 처리
3. 서버가 AG-UI 이벤트들을 순서대로 전송
4. 클라이언트(UI)가 이벤트를 받아 화면을 업데이트

핵심 포인트는 **"결과"가 아니라 "과정(이벤트)"도 함께 전달**된다는 점입니다.

## 4) 초보자용 팁

- **팁 1: 먼저 Dojo로 감 잡기**  
  코드 보기 전에 Dojo 데모를 직접 눌러보면 개념 이해가 빠릅니다.

- **팁 2: 처음엔 메시지 이벤트부터**  
  처음부터 모든 이벤트를 다루지 말고, 텍스트 메시지 흐름부터 확인하세요.

- **팁 3: 상태 이벤트는 나중에 확장**  
  기본 채팅이 동작한 뒤에 state snapshot/delta를 붙이면 덜 복잡합니다.

- **팁 4: 프레임워크보다 프로토콜 먼저**  
  특정 프레임워크 사용법보다, "이벤트를 어떻게 주고받는지"를 먼저 이해하면 응용이 쉬워집니다.

- **팁 5: 작은 기능 단위로 검증**  
  메시지 스트리밍 → 툴 호출 표시 → 상태 동기화 순서로 단계별 확인을 권장합니다.
